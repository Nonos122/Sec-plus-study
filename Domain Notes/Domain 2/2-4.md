# 2.4 Notes

## Malware

1. Ransomware — encrypts a victim's data and demands payment for the decryption key; often paired with "double extortion," threatening to leak stolen data too.
2. Trojans — malware disguised as legitimate software to trick a user into installing it.
3. Remote Access Trojan (RAT) — a trojan that gives an attacker persistent remote control over the infected system.
4. Worms — self-replicating malware that spreads across a network on its own, without needing a host file or user action.
5. Spyware — secretly monitors user activity (keystrokes, browsing, screen) and sends the data back to an attacker.
6. Bloatware — unwanted, pre-installed software that isn't necessarily malicious but consumes resources and can widen the attack surface.
7. Viruses — malicious code that attaches to a host file and requires user action (like opening the file) to execute and spread.
8. Polymorphic viruses — viruses that change their own code/signature each time they run, making them harder for signature-based antivirus to detect.
9. Keyloggers — malware or hardware that records every keystroke a user types, often used to steal credentials.
10. Logic bombs — malicious code that lies dormant until a specific condition or trigger (a date, an action, an event) is met.
11. Rootkits — malware that embeds itself deep in the OS or kernel to hide its presence and maintain privileged, persistent access.

**Added:** Botnet/bot — a network of compromised devices ("bots" or "zombies") controlled remotely by an attacker (via a command-and-control server) to carry out coordinated actions like DDoS attacks or spam campaigns. Worth knowing alongside malware types since botnets are usually built from worm- or trojan-infected machines.

Techniques used to analyze suspicious files and determine whether they're malicious:

1. Static analysis — examining a file's code, strings, and structure without executing it (e.g., reviewing disassembled code, checking file hashes against known malware signatures).
2. Dynamic analysis — running the file in a controlled, isolated environment and observing its actual behavior (network calls, file changes, registry edits).
3. Sandboxing — a specific form of dynamic analysis where the suspicious file is executed in an isolated virtual environment, safely separated from production systems, so its behavior can be observed without risk.

## Physical Attacks

1. Brute force — repeatedly attempting physical access (e.g., trying keys/codes) until access is gained.
2. RFID cloning — copying the data from a legitimate RFID badge/card onto a duplicate, allowing unauthorized physical access.
3. Environmental — attacks that manipulate a facility's environment (temperature, humidity, power, fire suppression) to cause damage or disruption to equipment.

## Network Attacks

1. DDoS attack — a Distributed Denial-of-Service attack that uses many sources (often a botnet) to overwhelm a target and take it offline.
2. Pivoting — using a compromised system as a launch point to move laterally and attack other systems on the same network.
3. Amplified DDoS — an attacker sends small requests to a third-party server that generates much larger responses directed at the victim, amplifying the attack's impact.
4. Reflected attack — an attacker spoofs the victim's IP address when sending requests to a third-party server, so the responses are "reflected" back to the victim instead of the attacker.
5. DNS attack — attacks targeting DNS, such as DNS poisoning (corrupting cache with false records) or DNS tunneling (hiding data/commands inside DNS traffic).
6. Wireless attack — attacks against Wi-Fi networks, such as evil twin access points, deauthentication attacks, or exploiting weak encryption.
7. On-path attack — an attacker positions themselves between two communicating parties to intercept or alter traffic (formerly called a man-in-the-middle attack).
8. Credential replay — capturing valid authentication credentials (or tokens) and reusing them later to gain unauthorized access.
9. Malicious code — any code written specifically to cause harm, gain unauthorized access, or disrupt systems.

## Application Attacks

1. Injection — inserting malicious input (like SQL or scripts) into an application to manipulate its behavior.
2. Buffer overflow — writing more data to a memory buffer than it can hold, overwriting adjacent memory and potentially allowing code execution.
3. Replay — capturing valid data transmissions (like a login request) and resending them to trick a system into granting unauthorized access.
4. Privilege escalation — exploiting a flaw to gain higher-level access than originally granted.
5. Forgery — tricking a user's browser into submitting an unwanted, authenticated request on their behalf (Cross-Site Request Forgery / CSRF).
6. Directory traversal — manipulating file path input to access files/directories outside the intended, restricted folder.

## Password Attacks

1. Password spraying — trying one common password against many accounts, avoiding lockouts triggered by repeated attempts on a single account.
2. Brute force — systematically trying every possible combination until the correct password is found.
3. Dictionary attack — trying a curated list of likely passwords (common words, leaked passwords) rather than every possible combination.
4. Hybrid — combining dictionary words with variations (numbers, symbols, common substitutions) to guess passwords more efficiently.
5. Online attack — password guessing performed directly against a live system/login, limited by lockout policies and rate limiting.
6. Offline attack — password guessing performed against a stolen password hash file, with no lockout limits since it's done outside the live system.

**Added:** Rainbow table attack — using a precomputed table of hash values to quickly reverse a stolen password hash back to its plaintext value; salting passwords is the primary defense against this.

## Cryptographic Attacks

1. Downgrade
   - SSL/TLS downgrade — forcing a connection to use an older, weaker version of SSL/TLS that's easier to break.
   - SSL stripping — intercepting a connection and forcing it to use unencrypted HTTP instead of HTTPS, so traffic can be read in plaintext.
2. Collision — finding two different inputs that produce the same hash output, undermining the integrity guarantees hashing is supposed to provide.
3. Birthday — a statistical attack that exploits the "birthday paradox" to find hash collisions faster than expected, given a large enough set of inputs.
4. Pass-the-hash attack — using a stolen password hash directly to authenticate, without ever needing to crack it back into plaintext.

## Indicators of Attack

1. Account lockout — an unusual number of accounts being locked out can indicate a brute-force or password-spraying attempt.
2. Concurrent session usage — the same account logged in from multiple locations/devices at once can indicate credential theft.
3. Blocked content — repeated attempts to access restricted resources may indicate reconnaissance or an active attack.
4. Impossible travel — a login from one location followed shortly by a login from a geographically distant location, faster than physically possible, indicates compromised credentials.
5. Resource inaccessibility — critical systems or files suddenly unavailable, possibly due to ransomware, a DoS attack, or sabotage.
6. Out-of-cycle logging — log entries appearing at unusual times (outside normal business/maintenance hours) can signal unauthorized activity.
7. Published/documentation — leaked internal documentation or credentials appearing publicly (e.g., on a paste site or dark web forum) is a strong indicator of compromise.
8. Missing logs — gaps or deletions in log data often indicate an attacker covering their tracks.

**Added:** Resource consumption — abnormally high CPU, memory, disk, or bandwidth usage on a system can indicate malware, cryptomining, or a DoS condition, and is commonly listed alongside resource inaccessibility as a key indicator.
