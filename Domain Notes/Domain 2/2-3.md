# 2.3 Notes

## Application Vulnerabilities

1. Memory injection — inserting malicious code directly into a running process's memory space so it executes with that process's privileges.
2. Buffer overflow — writing more data to a memory buffer than it can hold, overwriting adjacent memory and potentially allowing code execution.
3. Race conditions — a flaw where the outcome depends on the timing of events, allowing an attacker to exploit the gap between two operations.
4. TOC/TOU (Time-of-Check to Time-of-Use) — a specific type of race condition where a resource is checked (e.g., for permissions) and then used, but changes between the check and the use.
5. Malicious update — a legitimate-looking software update that has been tampered with to deliver malware instead of (or alongside) the intended fix.
6. Operating system vulnerabilities / web-based vulnerabilities — flaws in an OS or web application that can be exploited to gain unauthorized access or control.
7. SQL injection (SQLi) — inserting malicious SQL code into an input field to manipulate or extract data from a backend database.
8. Cross-Site Scripting (XSS) — injecting malicious scripts into a trusted website so they execute in another user's browser.

## Hardware Vulnerabilities

1. Firmware — outdated or unpatched low-level software on a device that can be exploited if vendors are slow to release updates.
2. End-of-life — hardware no longer supported or patched by the manufacturer, leaving known flaws unaddressed permanently.
3. Legacy — older hardware that may not support modern security features or standards.

## Virtualization Vulnerabilities

1. VM escape — an attacker breaks out of an isolated virtual machine to access the underlying host or other VMs on the same hypervisor.
2. VM sprawl — an uncontrolled proliferation of virtual machines that aren't properly tracked, patched, or decommissioned, increasing the attack surface.
3. Resource reuse — sensitive data left behind in shared resources (memory, storage) that could be accessed by another VM reusing that resource.

## Cloud-Specific Vulnerabilities

Risks unique to cloud environments, including misconfigured storage (e.g., a publicly exposed storage bucket), insecure APIs, weak identity and access management, lack of visibility into the provider's infrastructure, and confusion over the shared responsibility model (not knowing which security tasks are the customer's responsibility versus the provider's).

## Supply Chain Vulnerability

1. Service provider — a third party operating or supporting critical systems on your behalf; a compromise on their end can expose your organization.
2. Hardware provider — a supplier of physical equipment that could be tampered with or contain counterfeit/compromised components before it reaches you.
3. Software provider — a vendor whose product or update process could be compromised, delivering malicious code to every customer using it.

## Cryptographic Vulnerability

Weaknesses arising from poor cryptographic implementation, such as using outdated or broken algorithms (e.g., MD5, DES), weak or reused keys, improper key storage, missing certificate validation, or misconfigured encryption settings — all of which can allow attackers to bypass or break the intended protection.

## Misconfiguration Vulnerability

Security weaknesses caused by improperly configured systems, such as default settings left unchanged, unnecessary services left running, overly permissive access controls, or missing security patches — often the result of human error rather than a flaw in the software itself.

## Mobile Device Vulnerabilities

1. Side loading — installing an application from a source other than the official app store, bypassing the store's security review process.
2. Jailbreaking — removing manufacturer-imposed restrictions on a device (commonly iOS), which also removes many of its built-in security protections.

## Zero-Day Vulnerabilities

A vulnerability that is unknown to the vendor (and therefore unpatched) at the time it is discovered or exploited, giving defenders no advance warning and no available fix.
