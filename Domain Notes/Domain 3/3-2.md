# 3.2 Notes

## Infrastructure Considerations

Before deploying security controls, an organization needs to think through where devices sit in the network, how traffic flows between zones, what needs to be isolated, and how the design supports both security and business operations. Good infrastructure design makes security controls (segmentation, monitoring, access control) effective; poor design undermines them no matter how good the individual tools are.

## Device Placement

Deciding where a security device (firewall, IPS, load balancer, proxy) sits in the network path determines what traffic it can see and control. Placement is typically driven by the security zones and traffic flows being protected — for example, a firewall between the internet and a DMZ, or an IPS placed inline in front of critical servers so it can inspect and block traffic before it arrives.

## Security Zones

1. Segmentation — dividing the network into distinct zones (e.g., DMZ, internal, guest) so traffic between them can be controlled and inspected.
2. Data protection — applying stronger controls (encryption, access restrictions) to zones holding more sensitive data.
3. Access control — enforcing which users/systems can move between zones.
4. Monitoring and logging — watching traffic within and between zones to detect suspicious activity.
5. Isolation — keeping a zone separated so a compromise there doesn't automatically spread elsewhere.
6. Compliance — some zones exist specifically to meet regulatory requirements (e.g., isolating cardholder data for PCI DSS).
7. Incident containment — well-defined zones make it easier to contain a breach to a limited area.
8. Operational efficiency — zoning traffic appropriately can also improve performance by keeping relevant traffic local.
9. Defense in depth — zones are one layer among several; combined with other controls, they reduce reliance on any single point of failure.

## Attack Surface

1. Endpoint — every workstation, laptop, and mobile device is a potential entry point.
2. Network services — any service listening on the network can potentially be probed or exploited.
3. Ports and protocols — every open port or enabled protocol is a possible way in.
4. User accounts and credentials — every account is a potential target for compromise.
5. Third-party integrations — external systems/vendors connected to your environment extend your risk beyond your own systems.
6. Cloud services — cloud resources add exposure through APIs, misconfigurations, and shared infrastructure.
7. Human factor — employees can be tricked via social engineering, regardless of how strong the technical controls are.

### Minimizing the Attack Surface

1. Vulnerability assessment — regularly scanning for and identifying weaknesses.
2. Access control — limiting who/what can reach a given resource.
3. Network segmentation — reducing what an attacker can reach if they compromise one part of the network.
4. Single point of failure (elimination) — removing/reducing critical components whose failure or compromise would have outsized impact.
5. Security updates — keeping systems patched to close known vulnerabilities.
6. Strong authentication — using MFA and strong credential policies to make account compromise harder.
7. Regular auditing — periodically reviewing configurations, access, and logs to catch issues early.
8. Security awareness — training users to reduce the human factor as a weak point.

## Connectivity

1. Scalability — the network design must support growth in users, devices, and traffic.
2. Security — connectivity choices (VPN, direct connections, cloud links) must be secured appropriately for what they carry.
3. Redundancy — multiple paths/connections ensure availability if one link fails.
4. Complexity — more connectivity options and paths mean more to secure and monitor.
5. Remote work — connectivity must support secure access for users outside the traditional office network.

## Failure Modes

1. Fail-open — if a security device fails, it allows all traffic through, prioritizing availability over security.
2. Fail-closed — if a security device fails, it blocks all traffic, prioritizing security over availability.

## Device Attributes

1. Active devices — actively inspect and can take action on traffic (e.g., an IPS that blocks malicious packets).
2. Passive devices — monitor traffic without altering it (e.g., an IDS that only alerts).
3. Inline — the device sits directly in the traffic path, so all traffic must pass through it (required for active devices like an IPS).
4. Tap/monitor — the device receives a copy of traffic (via a network tap or mirrored port) without sitting in the direct path, so it can't block traffic in real time.

## Network Appliances

1. Proxy server — sits between clients and the internet, forwarding requests on their behalf.
   - URL filtering — blocking access to specific websites or categories of sites.
   - Content filtering — inspecting and blocking content based on its actual contents, not just the URL.
   - Web page caching — storing frequently accessed pages locally to improve performance and reduce bandwidth use.
2. Reverse proxy server — sits in front of one or more internal servers, forwarding external requests to them; hides the internal servers' details and can provide load balancing, caching, and additional security filtering (e.g., in front of a WAF).
3. IPS (Intrusion Prevention System) — sits inline and actively blocks malicious traffic in real time.
4. IDS (Intrusion Detection System) — passively monitors traffic and generates alerts, without blocking anything itself.
5. Load balancer — distributes traffic across multiple servers to improve performance and availability.
   - Least utilized host — sends the next request to whichever server currently has the lightest load.
   - Affinity (session persistence) — ensures a specific client's requests keep going to the same server, important for maintaining session state.
   - DNS round robin — a simple method of load distribution where DNS responses cycle through a list of server IP addresses in turn.
   - Session/scheduling — the overall method (algorithm) the load balancer uses to decide where to send each request.

## Port Security

1. Sticky MAC — a switch port security feature that learns and locks the first MAC address(es) seen on a port, preventing unauthorized devices from connecting to that port.
2. Disabling unused ports — physically or administratively shutting down switch ports that aren't in use, so they can't be used to plug in an unauthorized device.
3. 802.1X authentication — a port-based network access control standard that requires devices to authenticate before being granted network access.
4. Extensible Authentication Protocol (EAP) — a framework used within 802.1X to support various authentication methods (e.g., certificates, tokens, credentials).

## Firewall Types

1. WAF (Web Application Firewall) — filters HTTP/HTTPS traffic to protect web applications from attacks like SQL injection and XSS.
2. UTM (Unified Threat Management) — an all-in-one security appliance combining multiple functions (firewall, antivirus, IPS, content filtering) into a single device.
3. NGFW (Next-Generation Firewall) — adds deep packet inspection, application awareness, and identity-based rules on top of traditional firewall filtering.
4. Layer 4 firewall — filters traffic based on IP address, port, and protocol, without inspecting the actual application content.
5. Layer 7 firewall — inspects and filters based on the actual application-layer content and behavior (e.g., a WAF is typically a Layer 7 firewall).

## Secure Communication / Access

1. VPN (Virtual Private Network) — creates an encrypted tunnel over an untrusted network (like the internet), commonly used for secure remote access.
2. Remote access
   - SSH (Secure Shell) — provides encrypted command-line remote access to a system.
   - RDP (Remote Desktop Protocol) — provides remote graphical desktop access to a system.
3. Tunneling — encapsulating one type of traffic inside another, often to secure it or move it across an incompatible network.
   - TLS (Transport Layer Security) — used to encrypt traffic at the transport layer, commonly for tunneling and securing web/application traffic.
   - IPSec (Internet Protocol Security) — a suite of protocols that authenticates and encrypts traffic at the network layer.
     - AH (Authentication Header) — provides integrity and authentication of the packet, but no confidentiality (no encryption).
     - ESP (Encapsulating Security Payload) — provides confidentiality (encryption) as well as integrity and authentication.

   | IPSec Component | Header | Payload Data | Trailer |
   |---|---|---|---|
   | AH | Added (authenticates header + payload) | Authenticated, not encrypted | None |
   | ESP | Added (authenticates payload) | Encrypted and authenticated | ESP trailer (padding, next-header field) |

   | IPSec Mode | What's Protected | Typical Use |
   |---|---|---|
   | Transport mode | Only the payload of the original packet; original IP header stays intact. | Host-to-host communication within a trusted network. |
   | Tunnel mode | The entire original packet (header + payload), wrapped in a new IP header. | Site-to-site VPNs, gateway-to-gateway communication. |

   *Always-on VPN* — a VPN connection that's automatically established and maintained continuously (rather than requiring the user to manually connect), ensuring devices stay protected whenever they have network access.

## Software-Defined Wide Area Network (SD-WAN)

Uses software to intelligently and dynamically route traffic across multiple connection types (MPLS, broadband, LTE/5G), improving cost, performance, and resilience compared to a traditional, statically configured WAN.

## Secure Access Service Edge (SASE)

A cloud-delivered architecture that converges networking (SD-WAN) and security services (Secure Web Gateway, CASB, ZTNA, Firewall-as-a-Service) into a single unified service, simplifying secure access for distributed and remote users.

## Selection of Effective Controls

1. Preventive — controls designed to stop a risk from happening in the first place (e.g., a firewall blocking unauthorized traffic).
2. Detective — controls designed to uncover risks or incidents that have already occurred (e.g., an IDS alert or log review).

*Added — the remaining control types worth knowing alongside these two:*
3. Corrective — controls that fix a problem after an event has happened (e.g., restoring from backup after ransomware).
4. Deterrent — controls that discourage an attack without directly preventing it (e.g., warning signs, visible cameras).
5. Compensating — an alternative control used when the primary control isn't feasible (e.g., extra monitoring on a system that can't be patched yet).

When selecting controls, effectiveness generally comes from combining multiple types across the different categories (technical, managerial, operational, physical) rather than relying on any single control alone.
