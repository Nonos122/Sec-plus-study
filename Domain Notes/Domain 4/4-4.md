# 4.4 Notes

## Monitoring Computing Resources

1. Log files — records of events generated by systems, applications, and devices, forming the raw data used for monitoring and investigation.
2. Security logs — logs specifically capturing security-relevant events, like authentication attempts, permission changes, and policy violations.
3. System monitors — tools that track the health and performance of an OS or device (CPU, memory, disk, uptime).
4. Application monitors — tools that track the health, performance, and behavior of specific applications.
5. Infrastructure monitors — tools that track the health and status of the broader network and infrastructure (routers, switches, links).

### SNMP (Simple Network Management Protocol)

1. SNMP agents — software running on a managed device that collects and reports data about that device.
2. SNMP managers — the central system that queries agents and collects the reported data.
3. SNMP traps — unsolicited alerts sent by an agent to the manager when a significant event occurs, rather than waiting to be asked.
4. NMS (Network Management System) — the platform used to monitor and manage network devices via SNMP, aggregating data from many agents.

### NIDS (Network Intrusion Detection System)

1. Traffic analysis — inspecting network traffic to identify patterns matching known attacks or anomalies.
2. Alerts and notifications — generating alerts when suspicious activity is detected.
3. Passive role — NIDS monitors a copy of traffic (via a tap or mirrored port) and cannot block anything itself; it only detects and alerts.

### NIPS (Network Intrusion Prevention System)

1. Real-time analysis and action — inspects traffic inline and can respond immediately as threats are detected.
2. Blocking threats — actively drops or blocks malicious traffic in real time.
3. Policy threats — traffic that violates configured security policy can be flagged or blocked, not just known attack signatures.
4. Policy enforcement — actively ensures traffic complies with defined security rules.
5. Reducing attack surface — by blocking malicious/unauthorized traffic before it reaches its target.
6. Alerts and reporting — still generates alerts and logs for review, in addition to taking action.

## Activities

1. Log aggregation — collecting logs from many different sources into one centralized location for easier analysis.
2. Alerting — automatically notifying staff when monitored data indicates a potential issue or threat.
3. Scanning — actively probing systems to identify vulnerabilities, misconfigurations, or unauthorized changes.
4. Reporting — summarizing monitoring data and findings for stakeholders.

**Key elements:**

1. Customizable dashboards — visual displays that can be tailored to show the metrics and alerts most relevant to a given team or role.
2. Cisco DNA Center — an example of a centralized network management platform providing monitoring, automation, and policy enforcement across Cisco infrastructure.
3. Compliance reporting — generating reports that demonstrate adherence to regulatory or policy requirements.
4. Executive summaries — high-level, non-technical reports summarizing security posture and key findings for leadership.

### Archiving

Retaining logs and monitoring data for a defined period (per policy or regulatory requirement) so they remain available for later investigation, auditing, or compliance purposes, even after they're no longer actively needed for daily monitoring.

## Alert Response and Remediation/Validation

1. Quarantine — isolating a suspected compromised system to prevent it from causing further harm while it's investigated.
   - Automated response — the system automatically quarantines a device based on predefined rules, without waiting for human action.
   - Manual response — a security analyst decides to quarantine a device based on their own judgment.
   - Manual intervention — a human is required to review and approve an action before it's carried out.
   - Isolation duration — how long a device stays quarantined, whether until manually cleared or after a set period.
2. Alert tuning — adjusting detection rules and thresholds to improve accuracy over time.
   - False positive — an alert fires but there's no real threat.
   - False negative — a real threat occurs but no alert fires.
   - True positive — an alert correctly fires because there is a real threat.

## Tools

- SCAP (Security Content Automation Protocol) — a suite of standards for automating vulnerability management and policy compliance checking.
  - OVAL (Open Vulnerability and Assessment Language) — a standardized format for describing system configuration, vulnerability, and patch state.
  - XCCDF (Extensible Configuration Checklist Description Format) — a standardized format for writing and automating security checklists/benchmarks.

## Benchmarks

Documented, industry-accepted configuration standards (like CIS Benchmarks) used as a reference point to measure whether a system's actual configuration meets security best practices.

## Agent/Agentless

- Agent-based monitoring — requires software installed directly on the target system, providing deeper visibility (e.g., into local processes and files) at the cost of more deployment/maintenance overhead.
- Agentless monitoring — collects data remotely without installing anything on the target, easier to deploy at scale but often with less depth of visibility.

## SIEM (Security Information and Event Management)

1. Data collection — gathering log and event data from across the environment.
2. Data aggregation — consolidating that data into a centralized system for analysis.
3. SIEM correlation — analyzing events together (rather than in isolation) to identify patterns that indicate an attack, which might not be obvious from any single log entry.
4. Alerting and reporting — generating alerts on suspicious patterns and producing reports for analysts and stakeholders.

## Antivirus

Software that detects, blocks, and removes known malware, primarily using signature-based detection (matching files against a database of known threats), sometimes supplemented with basic heuristic/behavioral detection.

## Data Loss Prevention (DLP)

A set of tools and policies designed to detect and prevent sensitive data from leaving the organization improperly, whether through email, file transfers, removable media, or cloud uploads — by inspecting content and blocking or flagging policy violations.

## SNMP Traps

(See SNMP section above.) Unsolicited messages sent from an SNMP agent to the manager/NMS to immediately report a significant event, rather than waiting for the manager to poll for it — useful for near-real-time alerting on device status changes.

## NetFlow

A network protocol (originally developed by Cisco) that collects metadata about network traffic flows (source/destination, ports, volume, duration) without capturing full packet contents, useful for traffic analysis, capacity planning, and detecting anomalous communication patterns.

## Vulnerability Scanners

Automated tools that scan systems, networks, and applications to identify known vulnerabilities, misconfigurations, and missing patches, producing reports that feed into the broader vulnerability management process.
