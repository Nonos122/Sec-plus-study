# 4.1 Notes

## Secure Baselines

A secure baseline is a documented, approved starting configuration that all systems of a given type should match, closing off unnecessary settings and reducing inconsistency across the environment.

### Establish

- CIS Benchmark — a set of vendor-neutral, consensus-based configuration guidelines (from the Center for Internet Security) covering operating systems, applications, and devices.
- STIG (Security Technical Implementation Guide) — configuration standards published by the U.S. Defense Information Systems Agency (DISA), commonly required in government/military environments.

### Deploy

- Microsoft Group Policy — a Windows feature that lets administrators centrally configure and enforce settings across many computers/users in a domain.
- Puppet Forge — a repository of Puppet configuration modules used to automate the deployment and enforcement of system configurations at scale.

### Maintain

- SCAP (Security Content Automation Protocol) compliance checker — a standardized tool that automatically checks systems against a defined security baseline/policy.
- CIS configuration assessment tool — a tool used to scan systems and verify they conform to CIS Benchmark settings.

### Hardening Targets

- Mobile devices
- Workstations
- Switches
- Routers
- Cloud infrastructure
- Servers
- ICS & SCADA
- Embedded systems
- RTOS
- IoT devices
- Wireless access points

Each of these has its own baseline, since the appropriate hardening steps (disabling ports, patching, restricting access) differ based on the device's function and constraints.

## Wireless Devices

- Site survey — physically assessing a location to determine optimal access point placement, signal coverage, and potential sources of interference before deployment.
- Heat maps — visual representations (generated from a site survey) showing wireless signal strength across a physical space, used to identify coverage gaps or overlap.

## Mobile Solutions

### Mobile Management

- MDM (Mobile Device Management) — software that allows an organization to centrally enforce security policies, push configurations, and remotely wipe or lock managed mobile devices.

### Deployment Models

- BYOD (Bring Your Own Device) — employees use their personal devices for work; convenient and cost-saving, but harder to secure and enforce policy on.
- CYOD (Choose Your Own Device) — employees pick from a company-approved list of devices, which they may own personally, balancing choice with easier security management.
- COPE (Corporate-Owned, Personally Enabled) — the company owns the device but allows personal use, giving the organization full control while still offering employee flexibility.

## Connection Methods

- Captive portal — a web page that requires users to authenticate or accept terms before being granted network access (common on guest Wi-Fi).
- WPS (Wi-Fi Protected Setup) — a feature meant to simplify connecting devices to a wireless network, but known to have significant security weaknesses (PIN brute-forcing) and is often disabled.
- PSK (Pre-Shared Key) — a shared password used to authenticate to a wireless network, common in home and small-business Wi-Fi (e.g., WPA2/WPA3-Personal).
- Evil twin attack — a rogue access point set up to mimic a legitimate network's SSID, tricking users into connecting so their traffic can be intercepted.

### Bluetooth

- Authentication — verifying the identity of a device before allowing a Bluetooth connection/pairing.
- BLE (Bluetooth Low Energy) — a power-efficient version of Bluetooth commonly used in IoT and wearable devices, with its own distinct security considerations.
- Bluetooth attacks — includes bluejacking (sending unsolicited messages/data), bluesnarfing (unauthorized access to data on a device), and bluebugging (gaining full remote control of a device).

### Cellular

Mobile network connectivity (4G/5G) used by devices to communicate independently of Wi-Fi; provides broader coverage but is still subject to interception and location-tracking risks if not properly secured.

### GPS

Global Positioning System — used for location tracking on mobile devices; raises both security (device tracking, geofencing enforcement) and privacy considerations.

## Mobile Solutions — Other Factors

- RFID (Radio-Frequency Identification) — short-range wireless technology used for identification/tracking (e.g., badges); vulnerable to cloning if not properly secured.
- Geolocation — using a device's physical location to inform access decisions (e.g., blocking logins from unexpected countries) or enforce policy.
- Tethering — sharing a mobile device's cellular data connection with other devices, which can bypass corporate network controls if not managed.

## Wireless Security Settings

### Wi-Fi Protected Access 3 (WPA3)

- PMF (Protected Management Frames) — encrypts and authenticates management frames (not just data frames), protecting against certain deauthentication and spoofing attacks.
- WPA3-Enterprise — uses 802.1X and a RADIUS server for per-user authentication, with stronger cryptographic requirements than WPA2-Enterprise.
- SAE (Simultaneous Authentication of Equals) — the key exchange method used in WPA3-Personal, replacing the WPA2 four-way handshake and resisting offline dictionary attacks.
- Wi-Fi Easy Connect — simplifies securely provisioning devices without displays (like IoT devices) onto a network, often using QR codes.
- Wi-Fi Enhanced Open — provides encryption for open (password-less) networks using Opportunistic Wireless Encryption (OWE), improving on traditional unencrypted open Wi-Fi.

## AAA/RADIUS

- Authentication — verifies who a user or device is.
- Authorization — determines what an authenticated user or device is allowed to do.
- Accounting — logs details of activity (username, timestamps, data usage) for later review.
- RADIUS clients — network devices (like a wireless access point or VPN concentrator) that forward authentication requests to a RADIUS server.
- Shared secret — a pre-configured key used between a RADIUS client and server to authenticate and encrypt their communication with each other.

## Cryptographic Protocols

- WEP (Wired Equivalent Privacy) — the oldest Wi-Fi security protocol, using RC4 encryption; deprecated and easily broken.
- WPA (Wi-Fi Protected Access) — introduced TKIP to address WEP's weaknesses, but still considered outdated and insecure by modern standards.
- WPA2 — introduced AES-CCMP encryption, a major security improvement over WPA/WEP; widely used but vulnerable to offline dictionary attacks against the four-way handshake.
- WPA3 — the current standard, replacing the WPA2 handshake with SAE and adding stronger protections like PMF.

## Authentication Protocols

- PEAP (Protected EAP) — wraps EAP communication inside an encrypted TLS tunnel, protecting credentials during authentication.
- 802.1X — a port-based network access control standard requiring authentication before granting network access.
- EAP-TLS — uses digital certificates on both the client and server for mutual authentication; very secure but requires a full PKI deployment.
- EAP-TTLS — extends TLS to allow a wider range of authentication methods inside the encrypted tunnel, without requiring a client-side certificate.
- EAP-FAST — a Cisco-developed EAP method that uses a Protected Access Credential (PAC) instead of certificates to establish a secure tunnel.

## Application Security

- Input validation — checking and sanitizing user input to prevent injection attacks and malformed data from reaching the application.
- Secure cookies — cookies flagged (e.g., Secure, HttpOnly) to ensure they're only sent over encrypted connections and can't be accessed by client-side scripts.
- Static code analysis — reviewing an application's source code without executing it, to catch vulnerabilities early in development.
- Code signing — digitally signing software to verify its authenticity and confirm it hasn't been tampered with since it was signed.
- Secure coding practices — following established guidelines (e.g., avoiding hardcoded credentials, proper error handling, least privilege) throughout development to reduce vulnerabilities.

## Sandboxing

Running an application or file in an isolated environment, separate from production systems, so its behavior can be observed safely without risking the rest of the environment.

## Monitoring

Continuously observing systems, applications, and network traffic to detect abnormal behavior, policy violations, or active threats, feeding into tools like SIEM for centralized analysis and alerting.
