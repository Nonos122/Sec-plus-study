# 4.9 Notes

## Log Data

1. Firewall logs — record allowed and denied connections, useful for identifying blocked attack attempts or unexpected traffic patterns.
2. Application logs — record events generated by specific applications, such as errors, user actions, and authentication attempts within that app.
3. Endpoint logs — record activity on individual devices, including process execution, file changes, and local login attempts.
4. OS-specific security logs — record security-relevant events at the operating system level, such as account logins, privilege use, and policy changes (e.g., Windows Security Event Log, Linux auth logs).
5. IPS/IDS logs — record detected or blocked malicious traffic and the signatures/rules that triggered the alert.
6. Network logs — record general network activity such as connections, bandwidth usage, and device communication patterns.
7. Metadata — data about data (timestamps, file size, sender/recipient, geolocation) that can reveal important context even without looking at the actual content.

## Data Sources

1. Vulnerability scans
   - Credentialed — scans performed with valid login credentials, giving a deeper, more accurate view of vulnerabilities from inside the system.
   - Non-credentialed — scans performed without credentials, showing only what's visible externally, like an outside attacker would see.
2. Automated reports — scheduled or triggered reports generated by security tools summarizing findings, trends, or incidents without requiring manual compilation.
3. Dashboards — visual, often real-time displays of security metrics and alerts, giving analysts an at-a-glance view of the current security posture.
4. Packet captures — full recordings of raw network traffic, allowing analysts to examine exactly what was sent between systems, down to the individual packet.
