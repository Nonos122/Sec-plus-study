# 4.7 Notes

## Use Cases of Automation & Scripting

1. User provisioning — automatically creating accounts and assigning appropriate access when a new employee joins, based on their role.
2. Resource provisioning — automatically deploying servers, storage, or cloud resources based on predefined templates.
3. Guardrails — automated policies that prevent risky configurations or actions (e.g., blocking the creation of a publicly exposed storage bucket) before they can happen.
4. Security groups — automatically applying and updating group-based permissions rather than managing them manually one by one.
5. Ticket creation — automatically generating a help desk or incident ticket when a monitoring tool detects an issue.
6. Escalation — automatically routing an unresolved alert or ticket to the appropriate team/person after a set time or severity threshold.
7. Enabling/disabling services — automatically turning services on or off based on policy or detected conditions (e.g., disabling an account after too many failed logins).
8. Continuous integration (CI) — automatically building, testing, and validating code changes as they're submitted, catching issues (including security flaws) early.
9. Integrations/API — connecting different tools and systems together programmatically so they can share data and trigger actions in one another automatically.

## Benefits of Automation and Scripting

1. Efficient/time-saving — automating repetitive tasks frees staff to focus on higher-value work.
2. Enforces baselines — automated configuration ensures every system matches the approved secure baseline consistently.
3. Standardizes infrastructure configuration — reduces variation between systems, making the environment easier to secure and troubleshoot.
4. Secure scaling — allows an organization to grow its infrastructure without a proportional increase in manual security effort, since new systems inherit the same automated controls.
5. Employee retention — reducing tedious manual work can improve job satisfaction and reduce burnout.
6. Faster reaction time — automated detection and response can act in seconds, far faster than a human could manually respond.
7. Workforce multiplier — automation lets a smaller team accomplish what would otherwise require significantly more staff.

## Other Considerations

1. Complexity — automated systems and scripts add their own complexity, which must be documented and understood, or they become a liability themselves.
2. Cost — building, testing, and maintaining automation/scripting requires upfront and ongoing investment.
3. Single point of failure — a flaw or outage in a central automation system can affect everything it manages at once, rather than just one system.
4. Technical debt — quickly written scripts/automation that aren't properly maintained can accumulate problems over time, becoming harder and riskier to change later.
5. Ongoing supportability — automation needs to be maintained, updated, and understood by the team long-term, not just built once and forgotten.
